# DramaAdapt Skill 设计文档

> 将爆款小说网文改编为漫剧剧本的 Claude Code Skill

## 概述

DramaAdapt 是一个 Claude Code 用户级 Skill，用于将完整的网络小说文本自动改编为标准漫剧剧本。支持处理任意长度的小说（几万到百万字），通过5个阶段的流水线逐步生成人物小传、分集大纲和逐集详细剧本。

### 设计来源

基于 B站废才俱乐部 "干货教程！网文IP改编漫剧入门攻略"（BV1r7SMB2E5t）的方法论，结合漫剧行业标准编剧规范设计。

## 目标与范围

### 输入
- 完整小说文本文件（.txt），支持几万到百万字

### 输出
- 纯剧本文本：人物小传 + 分集大纲 + 逐集详细剧本
- 不含分镜/绘图描述

### 安装位置
- `~/.claude/skills/drama-adapt/SKILL.md`

## 整体架构

### 流程设计（5阶段 + 3确认节点）

```
启动向导(配置)
  → 阶段1: 原著分析与摘要
  → 阶段2: 人物小传生成 → [确认节点A]
  → 阶段3: 分集大纲生成 → [确认节点B]
  → 阶段4: 逐集剧本生成 → [确认节点C]
  → 阶段5: 全文质检与输出
  → 完成
```

### 运行模式
- **默认模式**：在3个确认节点暂停等待用户确认
- **自动模式** (`--auto`)：跳过所有确认节点，全程自动
- **续跑模式**：检测到 project.json 自动恢复上次进度
- **重跑模式** (`--from=N`)：从指定阶段重新执行

### 文件组织（在用户运行目录下生成）

```
小说项目文件夹/
├── novel.txt              # 用户放入的原著文件
├── project.json           # 项目状态与配置
├── analysis.md            # 阶段1: 原著分析报告
├── characters.md          # 阶段2: 人物小传
├── outline.md             # 阶段3: 分集大纲
├── scripts/               # 阶段4: 逐集剧本
│   ├── ep001.md
│   ├── ep002.md
│   └── ...
├── qc_report.md           # 阶段5: 质检报告
└── final_script.md        # 最终合并剧本（可选）
```

## 启动向导

### Step 1: 检测运行目录
- 搜索 txt 文件（`novel.txt` / `小说.txt` / 任意 .txt）
- 找到 → 预览前200字，统计总字数，继续 Step 2
- 找不到 → 提示用户放入小说文件后停止

### Step 2: 检测项目状态
- `project.json` 存在 → 显示进度，进入续跑模式
- 不存在 → 进入 Step 3 初始化问答

### Step 3: 初始化问答（3个配置问题）

**问题1：题材类型**
```
1. 都市言情（甜宠/虐恋/豪门）
2. 古代言情（宫斗/穿越/仙侠）
3. 都市爽文（逆袭/重生/赘婿）
4. 玄幻修仙（升级/热血/战斗）
5. 悬疑推理（烧脑/反转/刑侦）
6. 其他（自定义描述）
```

**问题2：剧本规格**
```
1. 短季（30集，每集60-90秒）← 推荐新手
2. 标准季（60集，每集60-90秒）
3. 长季（100集，每集60-90秒）
4. 自定义（手动设置集数和单集时长）
```

**问题3：改编策略**
```
1. 忠实改编（保留原著情节和人物）← 推荐
2. 精简改编（大幅精简支线，突出主线爽点）
3. 创意改编（在原著基础上增加原创情节/人物）
```

### 自动计算参数
| 参数 | 计算规则 |
|------|----------|
| 单集字数 | 60秒≈500-600字，90秒≈700-900字 |
| 付费卡点 | 默认第10集后 |
| 钩子密度 | 每集结尾必设悬念 |
| 爽点密度 | 每2-3集一个主要爽点 |

### project.json 结构
```json
{
  "version": "1.0",
  "created_at": "ISO时间戳",
  "novel_file": "novel.txt",
  "novel_chars": 350000,
  "settings": {
    "genre": "都市言情",
    "total_episodes": 30,
    "episode_duration": "60-90",
    "episode_script_chars": "500-900",
    "adaptation_strategy": "忠实改编",
    "paywall_after": 10
  },
  "stages": {},
  "characters": [],
  "episodes": []
}
```

## 阶段详细设计

### 阶段1：原著分析与摘要

**目的**：读取完整小说，生成结构化分析报告

**处理超长文本策略**：
1. 计算小说总字数
2. 分块读取（每块约3-5万字，保留块间重叠500字）
3. 每块提取：主要人物、关键事件、情节转折、爽点
4. 汇总所有块的分析结果，生成整体报告

**输出 `analysis.md`**：
```markdown
# 原著分析报告

## 基本信息
- 书名/标题：
- 总字数：xxx万字
- 主要情节线：N条
- 核心爽点数量：N个

## 故事主线概要（500-1000字）

## 主要人物列表
| 姓名 | 身份 | 性格特征 | 重要程度 |

## 关键情节节点（按时间线）
1. [事件描述] - [类型：爽点/转折/高潮]

## 改编建议
- 推荐保留的情节线
- 建议删减的支线
- 推荐的开篇位置
- 推荐的付费卡点位置
```

完成后更新 project.json stages 状态，自动进入阶段2。

### 阶段2：人物小传生成

**目的**：基于分析报告，生成每个主要人物的详细小传

**输出 `characters.md`**：
每个角色包含：
- 姓名、年龄、外貌（3-5句详细描写）
- 性格（核心特质 + 描述）
- 背景（家庭、经历）
- 目标/动机（角色驱动力）
- 人物弧线（从A到B的成长变化）
- 标志性台词风格
- 与其他角色关系

**确认节点A**：
> 人物小传已生成，请检查是否符合你的理解。回复【确认】继续，或回复修改意见。

### 阶段3：分集大纲生成

**目的**：将原著情节拆分为指定集数的分集大纲

**拆分原则**：
1. 每集聚焦一个核心事件/冲突
2. 每集结尾必须有悬念钩子
3. 前10集（免费区）密集爽点
4. 第10集结尾设置强钩子（付费卡点）
5. 每3-5集安排一个情感高潮/重大转折
6. 最后一集收束主线

**输出 `outline.md`**：
每集包含：
- 集名
- 核心事件（一句话）
- 关键场景列表
- 出场人物
- 爽点/钩子描述
- 情绪曲线

**确认节点B**：
> 分集大纲已生成。请重点检查情节拆分、付费卡点位置、爽点密度。

### 阶段4：逐集剧本生成

**目的**：根据大纲逐集生成详细剧本

**剧本格式规范**：
```markdown
# 第X集：[集名]

## 场1-1 日/内 客厅

**画面**：[场景环境描写，2-3句]

**[角色A]**
（动作/神态描写）
对白内容

---

## 场1-2 夜/外 街道

OS（角色A内心独白）：内心想法

---

【本集钩子】
[结尾悬念描述]
```

**质量控制**：
- 字数控制在设定范围内
- 对白精简，单句不超过2行
- 每集场景5-8场
- 动作描写用现在时态

**批量生成策略**：
- 每次生成5集，检查连贯性后继续
- 回读前一集结尾确保衔接

**确认节点C**：
> 全部剧本已生成，请抽查几集检查质量。

### 阶段5：全文质检与输出

**三轮自动质检**：
1. 格式质检：场景头、角色名一致性、字数范围
2. 叙事质检：情节连贯性、时间线、逻辑漏洞
3. 角色质检：性格一致性、台词风格、关系变化

**输出**：
- `qc_report.md`：质检报告
- `final_script.md`：合并剧本（可选）

## 高级命令

```
/drama-adapt                 ← 常规：续跑或新建项目
/drama-adapt --auto          ← 无人值守：跳过所有确认节点
/drama-adapt --from=3        ← 从阶段3重跑
/drama-adapt --stage=4       ← 仅重跑阶段4
```

## 技术约束

1. 纯 Claude 执行，不依赖外部脚本或 API
2. 所有输出为 Markdown 文件
3. 通过 project.json 实现状态持久化和续跑
4. 分块处理支持任意长度小说输入
